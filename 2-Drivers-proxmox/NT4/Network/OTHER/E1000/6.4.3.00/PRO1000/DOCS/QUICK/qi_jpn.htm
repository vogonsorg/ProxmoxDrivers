インテル(R) PRO ネットワーク アダプタのインストール

2004 年 4 月改訂

Intel PCI、PCI-X、または PCI-Express* アダプタのインストール

  1. コンピュータの電源を切り、電源コードをコンセントから抜きます。

  2. コンピュータのカバーを外し、アダプタに該当するスロットからアダプタ スロット カバーを取り外します。

  3. PCI Express アダプタの場合: アダプタのエッジ コネクタを PCI Express スロットに差し込み、ブラケットをシャーシに止めつけます。
    PCIおよび PCI-X アダプタの場合: アダプタのエッジ コネクタを、PCI スロットに完全にはまるまで差し込みます。 ブラケットをシャーシに止めつけます。

  4. コンピュータのカバーを取り付けます。 電源コードをコンセントに挿入します。

ネットワーク ケーブルの挿入

Intel(R) PRO/100 アダプタおよび PRO/1000 銅アダプタ 10 Mbps ネットワークでは、カテゴリ 3、4、または 5 のツイスト ペア イーサネット (TPE) ケーブルを使用して、ネットワーク ケーブルまたはケーブルをアダプタに接続します。 100 または1000 Mbps ネットワークおよびホーム ネットワークには、カテゴリ 5 (4 ペア) ケーブルを使用します。

Intel(R) PRO/1000 光ファイバ アダプタ (SC コネクタ接続): 光ファイバー コネクタのカバーをはずして保管します。 1000Base-SX 二重通信型 SC 光ファイバー コネクタを アダプタ ブラケットの TX/RX ポートに差し込みます。

Intel(R) PRO/1000 光ファイバ アダプタ (LC コネクタ接続): 光ファイバー コネクタのカバーをはずして保管します。 アダプタのタイプ (1000BASE-SX または 1000BASE-LX) に合ったケーブルを使用して、LC コネクタをアダプタに差し込みます。 長さの制限を含め、光ファイバーの仕様にケーブルが準拠する場合は、SC 等のほかのタイプのコネクタに変換ケーブルを使用することができます。

 コンピュータを起動し、以下のドライバ インストール手順に従います。

ドライバおよび拡張子のインストール

注: Intel PRO/1000 サーバ アダプタは、Windows 95*、Windows 98*、および Windows Me* のコンピュータでは動作しません。

Windows オペレーティング システム (Windows NT* 4.0 を除く)

ドライバをインストールするには、オペレーティング システムに対して管理者権限が必要であることにご留意ください。

  1. アダプタが検出され、Windows は Windows の搭載ドライバをインストールするか、新しいハードウェアの検出ウィザードを開始します。

  2. ウィザードが開始する場合、プロンプトに従い、このパッケージの CD からドライバをインストールします。

  3. Windows の搭載ドライバがインストールされる (ウィザードが表示されない) 場合、インテルの CD を挿入して、Intel PRO ネットワーク接続 オートランのウィンドウが表示される際に [ドライバのインストール] ボタンをクリックしてドライバを更新します。

  4. [インストールのオプション] ダイアログが表示されたら、チームまたは VLAN をセットアップする必要がある場合に [Advanced Network Services] を選択します。

Windows NT 4.0

注: インテル PROSet Utility ソフトウェアは、アダプタ ドライバのインストール中に自動的にインストールされます。

  1. Windows NT 4.0 を起動して、[コントロール パネル] で [ネットワーク] アイコンをダブルクリックし、 [アダプタ] タブ、[追加] の順にクリックします。

  2. アダプタのリストが表示されたら、そのリストからアダプタを選択せずにアダプタに搭載された CD を挿入するか、その他のメディアを示して、[ディスク使用] をクリックします。

  3. CD の [Autorun] の画面が表示されたら、それを閉じて続行します。

  4. パスを指定し [OK] をクリックしてインストールを続行します。 作業が終了すると、ネットワーク アダプタのリストに、インストールしたインテルのアダプタが表示されます。

  5. [閉じる] をクリックしてインストールを完了します。

  6. コンピュータを再起動します。

Linux*

このドライバのバイナリ RPM* パッケージをビルドするには、'rpmbuild -tb <ファイル名.tar.gz>' を実行します。 <ファイル名.tar.gz> をドライバの特定ファイル名に置き換えます。

  注: ビルドが適切に動作するには、現在実行しているカーネルがインストール済みのカーネル ソースのバージョンおよび設定と一致する必要があります。 カーネルを再コンパイルした直後の場合、システムを今すぐ再起動します。

RPM 機能は Red Hat 配布でのみテストされています。

インテル PRO/100 アダプタには、ドライバ名に e100 を使用します。

  1. 任意のディレクトリにベース ドライバの tar ファイルを移動します。 たとえば、'/home/username/e1000' または '/usr/local/src/e1000' を使用します。

  2. アーカイブを tar または zip 解除します。<x.x.x> はドライバ tar ファイルのバージョン番号です。

    tar zxf e1000-<x.x.x>.tar.gz

  3. ドライバ src ディレクトリに変更します。<x.x.x> はドライバ tar のバージョン番号です。

    cd e1000-<x.x.x>/src/

  4. ドライバ モジュールをコンパイルします。

    make install

    バイナリは次の通りにインストールされます。

    /lib/modules/<kernel_version>/kernel/drivers/net/e1000.o

    上記に示されたインストールの場所はデフォルトであり、 一部の Linux 配布では正しくない可能性があります。 詳細は、ldistrib.htm をご覧ください。

  5. モジュールをインストールします。

    insmod e1000 <パラメータ>=<値>

  6. 次のように入力して、IPアドレスをインターフェースに割り当てます。<x> はインターフェースの番号です。

    ifconfig eth<x> <IP_address>

  7. インターフェースが動作することを確認します。 次のように入力します。<IP_address> は、テストされるインターフェースとして同じサブネット上の別のコンピュータの IP アドレスです。

    ping <IP_address>
